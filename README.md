# 项目介绍



基本介绍
首先我是要写一个平台，用于小白AI生成的html网页上传，并添加一份"登录，导入导出local记录"功能。
这样的话，小白A提交html-a，其他用户访问a.shuijiaowang.com时，我会返回html-a+<登录/导入/导出.js脚本>，让用户ABCD可以实现数据存储，而小白A不需要写后端接口。
任务拆分
服务器
初始化服务器，宝塔面板，域名购买，后端初始化，管理端，数据库设计。
服务器:阿里云99/年2 核（vCPU）2 GiB,40G盘3MB固定宽带.
数据库基本设计
用户表，普通用户和管理端用户共享。
用户id，昵称，邮箱，密码。
html表-上传，管理端，用户小白上传
htmlid，用户id，html文本，子域名，简介，审核是否通过（交给AI）
用户数据表，存储local数据
用户id，htmlid，local数据json
一些问题？
如图片 
如果小白A上传的html引用了网页B的图片链接/加载库脚本，会发生什么情况？
也就是a.shuijiaowang.com页面中图片的src=bilibili.com/img/1
逻辑设计
上传者进入upload.shuijiaowang.com页面，看到一些介绍（说明介绍提示词修改QQ群建议共创），看到一些其他人写的网页（介绍，访问量）。
点击注册/登录，输入邮箱（用户名），密码，随机分配昵称为用户id，可以在个人中心点击修改昵称（唯一）
点击上传html文件，填写简介（可空），填写域名（可空）。
后端接收html文件，AI审核（暂时跳过），存入数据库中。
上传者在upload.shuijiaowang.com可以查看/删除/更新自己的内容（html文件，简介，域名等）
如a.shuijiaowang.com,简介：这是a写的html。域名：a
当用户B访问a.shuijiaowang.com时，后端返回a.html+注入脚本（悬浮按钮/登录/导入/导出/自动同步），如果是已登录状态会同时返回数据。数据同步在数据库中


后续可以做增值服务，如专属域名，如数量限制，如广告分成。
